üìã Project Overview

Business Turku is an MVP-level AI tool designed to revolutionize how companies discover and connect with funding opportunities. Based on a company name or business ID, the platform quickly generates a comprehensive company summary, identifies suitable public funding instruments, and matches companies with relevant investors using advanced semantic search and AI-powered analysis.

üéØ Core Value Proposition

Eliminate the manual research burden of finding suitable investors and funding programs. Our platform leverages AI and vector similarity search to instantly match companies with the most relevant funding opportunities and investors based on industry, stage, location, and investment criteria.

‚ú® Key Features

1 Company Search & Profiling

Users can search for companies by name or business ID. The system retrieves comprehensive company data from the Finnish Patent and Registration Office (PRH) API, including size, industry classification, registration details, and business information. Additional data points can be manually enriched by users.

2 Rich Company Data Display

Companies are presented with all available data from PRH API including business ID, name, industry codes, registration date, and address. Users can add missing information such as funding round, valuation, company hierarchy, and funding purpose to enhance matching accuracy.

3 AI-Powered Investor Matching

Using vector embeddings and semantic similarity search, the platform identifies and ranks investors (VCs, public investors, and public funding programs) that best match the company's profile. Matches are based on preferred industries, business models, investment rounds, geographic focus, check size, and investment thesis.

4 Intelligent Pitch Generation

For each matched investor, users can generate a customized pitch using OpenAI GPT models. The AI creates compelling, investor-specific pitches that highlight the alignment between the company's profile and the investor's preferences, portfolio, and investment criteria.

5 Interactive AI Pitch Refinement

Users can engage in a conversational interface with the LLM to iteratively refine and improve the pitch. The AI can adjust tone, emphasize different aspects, incorporate feedback, and tailor the message to specific investor preferences‚Äîall through natural language conversation.

6 Comprehensive Justifications

Each investor recommendation includes detailed justification explaining why they're a good match, including industry alignment, stage fit, geographic preferences, previous portfolio investments, and investment thesis compatibility.

System Components

Frontend Layer

  • Framework: Next.js (React-based)
  • Deployment: Vercel (business-turku-frontend-final.vercel.app)
  • Features: Company search interface, investor matching display, pitch generation UI, conversational AI interface

Backend Services

  • Runtime: Node.js
  • Data Loading Scripts: Automated pipelines for company and investor data ingestion
  • APIs: RESTful endpoints for search, matching, and pitch generation

Database & Storage

  • Platform: Supabase (PostgreSQL + pgvector extension)
  • Tables: company, company_embeddings, investor, investor_embeddings
  • Vector Search: HNSW indexes for fast similarity search with 1024-dimensional embeddings
  • Connection: postgresql://postgres:BusinessTurku@db.rvnljjfpwbvtcqsmedjg.supabase.co:5432/postgres

AI & ML Services

  • Embeddings: Voyage AI (voyage-2 and voyage-3.5 models, 1024 dimensions)
  • Pitch Generation: OpenAI GPT models (GPT-4/GPT-3.5-turbo)
  • Similarity Search: Cosine similarity via pgvector

External Data Sources

  • PRH API: Finnish Patent and Registration Office open data (avoindata.prh.fi)
  • Statistics Finland: Industry classification codes (toimiala_1_20250101)
  • DeepL API: Finnish-to-English translation for industry categories

Data Processing Pipeline

Company Data Pipeline

  • Fetch companies from PRH API with pagination (up to 2000 companies)
  • Filter for companies with industry classifications
  • Translate Finnish industry category names to English using DeepL API
  • Store company data in Supabase (bulk load without embeddings for performance)
  • Optional: Generate vector embeddings for semantic search using Voyage AI

Investor Data Pipeline

  • Load investor profiles from structured JSON file (20 investors)
  • Store complete investor data in JSONB field
  • Extract 7 key attributes for vectorization: preferred_industries, business_models, preferred_rounds, geo_focus, check_size_range, investment_thesis, avoid_industries
  • Generate 1024-dimensional embeddings using Voyage AI (voyage-2 model)
  • Store embeddings in investor_embeddings table with HNSW index

Matching & Ranking Algorithm

  • Create company text from name, business ID, industry, description, and address
  • Generate vector embedding for company query
  • Perform cosine similarity search against investor embeddings
  • Rank investors by similarity score
  • Filter by user preferences (region, investor type, stage)
  • Generate justifications highlighting matching attributes

üìä Data Models

Company Data Attributes

  • From PRH API: Business ID, Name, Industry Classification Code, Registration Date, Address, Business Status
  • User Enriched: Size (EU standards), Valuation, Current Investment Round, Company Hierarchy, Funding Size & Purpose
  • Computed: Industry Category Name (translated to English), Company Description

Investor Data Attributes

  • Identity: Name, Role, Firm, Location, Active Status
  • Investment Criteria: Preferred Industries, Business Models, Preferred Rounds, Round Role, Check Size Range
  • Geographic: Geo Focus (regions of investment)
  • Strategy: Investment Thesis, Avoid Industries
  • Track Record: Portfolio Companies, Previous Investments, Background
  • Engagement: Evaluation Criteria, Personality Type, Support Level, Contact Policy

Matching Attributes (Used for Vector Embeddings)

  • preferred_industries - Industry sectors of interest
  • business_models - Preferred business models (B2B, B2C, SaaS, etc.)
  • preferred_rounds - Investment stages (Pre-Seed, Seed, Series A, etc.)
  • geo_focus - Geographic regions for investment
  • check_size_range - Ticket size range
  • investment_thesis - Investment philosophy and focus areas
  • avoid_industries - Industries explicitly excluded

üìã Functional Requirements

Input and Data Collection Requirements

FR-01: The system must allow users to manually input company data fields, including name or business ID, and retrieve information such as size, field of work, valuation, investment round, hierarchy, funding purpose, and overall company description.
FR-02: The system must retrieve or validate company data using the PRH API.
FR-03: The system must store investor attributes including name, region, major fields, and previous investments.
FR-04: The system must maintain structured metadata for public funding programs, including eligibility criteria and deadlines.

Matching and Recommendation Requirements

FR-05: The system must generate a ranked list of suitable public funding instruments based on company eligibility. The system must generate a ranked list of suitable investors based on previous investment patterns and region.
FR-06: The system must provide justification text for each recommendation, including matching attributes (e.g., industry match, funding stage match).
FR-07: The system must allow users to filter recommendation results by region, investor type (public/VC), and investment stage.

User Interface Requirements

FR-08: The system must provide a single input form for entering or validating company information.
FR-09: The UI must present ranked funding and investor recommendations in separate sections.
FR-10: Each recommendation must include a summary card with justification and source links.

System Integration Requirements

FR-11: The system architecture must support future integration with MS Dynamics CRM.
FR-12: The system must support REST-based API communication for saving and retrieving company profiles.

Performance and Scalability Requirements

FR-13: System response time must allow ranking and justification generation within 5 seconds per request (preferred) for MVP.

Security and Access Requirements

FR-14: Company and investor metadata must be stored securely and comply with GDPR.
FR-15: Audit logs must track final outputs and changes when editing company profiles.
